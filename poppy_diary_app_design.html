Suggested tech stack
Recommended
Backend — Supabase
· PostgreSQL database (hosted)
· Auth with email/password & magic link
· Row-level security per user
· Storage bucket for photos
· Real-time subscriptions (optional)
· Free tier is generous
Alternative
Backend — Firebase
· Firestore (NoSQL, easier querying)
· Firebase Auth built-in
· Cloud Storage for photos
· Slightly more complex security rules
· Better for very fast prototyping
Mobile — React Native + Expo
· Single codebase for iOS + Android
· Expo Router for navigation
· expo-secure-store for PIN lock
· expo-image-picker for photos
· expo-file-system for import/export
· AsyncStorage for offline cache
Extras
· react-native-mmkv — fast local settings
· Zustand — lightweight state
· react-native-reanimated — smooth anims
· date-fns — date formatting
· react-hook-form — forms
· expo-crypto — local encryption
Database structure (Supabase / PostgreSQL)
users (managed by Supabase Auth)
  id · email · created_at

profiles
  id · user_id · theme · pin_enabled · pin_hash

entries
  id · user_id · title · content · color_tag
  word_count · created_at · updated_at

photos
  id · entry_id · user_id · storage_path · order_index

Row-level security: every table filters by auth.uid() = user_id
Feature flow — how to build each piece
PIN / app lock expo-secure-store saves the hashed PIN locally. On app launch, check store — if PIN exists, show lock screen before navigating. Never store plain PIN.
Photo handling expo-image-picker → compress locally → upload to Supabase Storage bucket → store path in photos table. On display, fetch signed URL. Photos render in their own horizontal strip, completely separate from text content area.
Import / export Export: fetch all entries → serialize to JSON (or Markdown per file) → expo-sharing. Import: pick file → parse → batch insert. Photos export as a zip with expo-file-system.
Search Supabase full-text search on title + content columns (PostgreSQL tsvector). Date filter uses created_at range. Color filter is a simple equality on color_tag. All filters compose.
Themes A ThemeContext holds the current theme name. Each theme is a JS object with 4–5 color tokens (accent, surface, light tint, text tint). Stored in profiles table, synced on login.
